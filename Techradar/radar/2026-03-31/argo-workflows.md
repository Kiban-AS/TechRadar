[Argo Workflows](https://argoproj.github.io/argo-workflows/) is an open-source, container-native workflow engine for Kubernetes designed to orchestrate parallel and sequential jobs. It is used to standardize cluster task automation and event-driven operations, providing a declarative way to define multi-step workflows as Kubernetes resources. Argo Workflows has been introduced to replace ad-hoc scripting with structured, auditable, and observable automation — including patterns for triggering workflows based on file changes and events from Azure Storage Queue. It is worth trialling for teams that need repeatable and scalable automation beyond what CI/CD pipelines can offer.
