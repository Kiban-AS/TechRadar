[Puppet](https://www.puppet.com/) is a configuration management tool that uses a declarative language to define the desired state of infrastructure. Puppet has a long history in infrastructure automation and has been used for provisioning and configuration management across server fleets. However, with the rise of cloud-native infrastructure and tools like Terraform, Bicep, Ansible, and Kubernetes-native configuration management, Puppet's model of managing long-lived, mutable servers has become less relevant. Puppet should not be used for new infrastructure automation projects. Teams still relying on Puppet should plan migration to modern IaC tooling such as Terraform, OpenTofu, or Ansible.
